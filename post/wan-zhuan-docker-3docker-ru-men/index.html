运行

docker run -i -t ubuntu /bin/bash
#-i:保持STDIN打开，即使没有连接#stdin是标准输入，一般指键盘输入到缓冲区里的东西。
#-t:分配一个pseudo-TTY
启动过程：

启动过程：

  1. ​ 检查本地是否存在 ubuntu镜像
  2. ​ 官方Docker Hub下载ubuntu镜像
  3. ​ Docker-Service使用镜像创建新容器

注意：Docker两种查看帮助方式

  • ​ man docker-help,man docker-ps
  • ​ docker help run,docker help ps

使用容器

容器的使用就相当于使用一台Linux主机，内部包含正常的网络，linux配置文件,也存在一个文件系统

root@zhang-Ubuntu:~# docker ps
CONTAINER ID        IMAGE               COMMAND             CREATED             STATUS              PORTS               NAMES
root@zhang-Ubuntu:~# docker ps -a
CONTAINER ID        IMAGE               COMMAND             CREATED              STATUS                          PORTS               NAMES
e3f664103613        ubuntu              "/bin/bash"         About a minute ago   Exited (4) About a minute ago                       sharp_solomon
root@zhang-Ubuntu:~# docker ps -al
CONTAINER ID        IMAGE               COMMAND             CREATED             STATUS                          PORTS               NAMES
e3f664103613        ubuntu              "/bin/bash"         2 minutes ago       Exited (4) About a minute ago                       sharp_solomon
  • Docker ps :查看正在运行容器的相关信息
  • Docker ps -a:查看当前系统的容器清单
  • docker ps -al:查看最后运行的一个容器

容器命名

Docker会为我们创建的每一个容器自动生成一个随机的名称。

命名规则：

  • ​ 容器的命名必须是唯一- 的。
  • ​ 小写字母az、大写字母AZ、数字0~9、下划线、圆点、横线(如果用正则表达式来表示这些符号，就是[a-z A-Z 0-9 _ . -])。
root@zhang-Ubuntu:~# docker run -i -t --name test_ubuntu ubuntu /bin/bash
root@zhang-Ubuntu:~# docker ps -al
CONTAINER ID        IMAGE    COMMAND       NAMES
ed5829718581        ubuntu   "/bin/bash"   test_ubuntu

--name 用户名 ：用来指定容器名称，此名称是用来方便判断容器用途。不是容器运行系统的主机名

重启已停止容器

容器启动:

  1. ​ docker start {CONTAINER ID}
  2. ​ docker start { NAMES}

容器重启:

  1. ​ docker restart {CONTAINER ID}
  2. ​ docker restart { NAMES}

容器只创建不运行:

root@zhang-Ubuntu:~# docker create -i -t --name helloworld ubuntu /bin/bash
c8f27daf22240940621372335644961142fc7b20a098860b96848ef39fac7f86

附着到容器

Docker容器重新启动的时候，会沿用docker c2 run命令时指定的参数来运行，因此我们的容器重新启动后会运行一个交互式会话shell。此外，也可以用docker attach命令，重新附着到该容器的会话上。

root@zhang-Ubuntu:~# docker start helloworld 
helloworld
root@zhang-Ubuntu:~# docker ps
CONTAINER ID        IMAGE               COMMAND             CREATED             STATUS              PORTS               NAMES
c8f27daf2224        ubuntu              "/bin/bash"         2 minutes ago       Up 9 seconds                            helloworld
root@zhang-Ubuntu:~# docker attach helloworld 
root@c8f27daf2224:/# exit

创建守护式容器

除了这些交互式运行的容器(interactive container)，也可以创建长期运行的容器。守护式容器(daemonized container)没有交互式会话，非常适合运行应用程序和服务。

docker run --name daemon_dave -d ubuntu /bin/sh -c "while true ;do echo hello world ; sleep 1; done"

root@zhang-Ubuntu:~# docker ps
CONTAINER ID   IMAGE      COMMAND                   NAMES
df18a8ce94a2   ubuntu     "/bin/sh -c 'while t…"    daemon_dave

docker run -d 命令使用了-d参数，因此Docker会将容器放到后台运行。

查看容器运行时日志

#本质上就是一个监控bash输入输入什么命令在屏幕上

docker logs 命令来获取容器的日志

docker logs daemon_dave #查看容器运行日志

docker logs -f daemon_dave #持续查看容器运行日志

docker logs --tail 10 daemon_dave #查看容器运行日志最后10行

docker logs --tail 0 -f daemon_dave #查看容器最新运行日志

docker logs -ft daemon_dave #持续查看容器运行日志，并加上时间戳

Docker日志驱动

Syslog:禁用docker的logs功能，将日志重定向到syslog

  • Docker run --log-driver="syslog"....:
  • Docker start --log-driver="syslog"....:

none：禁用所有容器中的日志，包括docker的logs功能

  • Docker run --log-driver="none"....:
  • Docker start --log-driver="none"....:

查看容器内进程

除了容器的日志，也可以查看容器内部运行的进程。

docker top 容器名

root@zhang-Ubuntu:~# docker top helloworld 
UID   PID           PPID   TTY      TIME         CMD
root  5068          5040   pts/0    00:00:00     /bin/bash

Docker统计信息

除了docker top命令，还可以使用docker stats 命令，它用来显示一一个或多个容器的统计信息。

root@zhang-Ubuntu:~# docker stats helloworld 
CONTAINER ID        NAME           CPU %     MEM USAGE / LIMIT     MEM %    
c8f27daf2224        helloworld     0.00%     3.195MiB / 4.297GiB   0.07%

客户端运行容器内部进程

在Docker 1.3之后，也可以通过docker exec命令在容器内部额外启动新进程。

  1. 在容器中运行后台任务

    root@zhang-Ubuntu:~# docker exec -d daemon_dave touch /abcd_file.txt
root@zhang-Ubuntu:~# docker exec -t -i daemon_dave ls -l /abcd_file.txt
-rw-r--r-- 1 root root 0 Apr  5 15:43 /abcd_file.txt
#这里的-d标志表明需要运行一一个后台进程，-d标志之后，指定的是要在内部执行这个命令的容器的名字以及要执行的命令。

    注意：从Docker 1.7开始，可以对docker exec 启动的进程使用-u标志为新启动的进程指定一个用户属主。

  2. 在容器内运行交互命令

    root@zhang-Ubuntu:~# docker exec -t -i daemon_dave ls -l /abcd_file.txt
#和运行交互容器时一样，这里的-t和-i标志为我们执行的进程创建了TTY并捕捉STDIN。

停止守护式容器

要停止守护式容器，只需要执行docker stop （容器名|ID ） 命令.

注意：

​ docker stop 命令会向Docker容器进程发送SIGTERM信号。如果想快速停止某个容器，也可以使用docker kill 命令来向容器进程发送SIGKILL信号。

docker ps -n X ：该命令会显示最后X个容器，不论这些容器正在运行还是已经停止，

自动重启容器

如果由于某种错误而导致容器停止运行，还可以通过--restart标志，让Docker自动重新启动该容器。--restart标志会检查容器的退出代码，并据此来决定是否要重启容器。默认的行为是Docker不会重启容器。

过滤出指定容器信息：

root@zhang-Ubuntu:~# docker inspect -f='{{.NetworkSettings.Gateway}}' daemon_dave
172.17.0.1

root@zhang-Ubuntu:~# docker inspect --format='{{.NetworkSettings.Gateway}}' daemon_dave
172.17.0.1
#--format 或者-f标志远非表面看上去那么简单。该标志实际上支持完整的Go语言模板。用它进行查询时，可以充分利用Go语言模板的优势
#--format 可以同时查看多个容器

注意：

除了查看容器，还可以通过浏览/var/lib/docker目录来深入了解Docker 的工作原理。该目录存放着Docker镜像、容器以及容器的配置。所有的容器都保存在/var/lib/docker/containers目录下。

删除容器

root@zhang-Ubuntu:~# docker rm test_ubuntu
test_ubuntu
#从Docker 1.6.2 开始，可以通过给docker rm 命令传递-f标志来删除运行中的Docker容器。这之前的版本必须先使用docker stop 或docker kill 命令停止容器，才能将其删除。
#删除所有容器：
docker rm -f  `docker ps -a -q `
