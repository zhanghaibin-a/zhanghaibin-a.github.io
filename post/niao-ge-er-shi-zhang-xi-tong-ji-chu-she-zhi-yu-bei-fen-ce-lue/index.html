备份要点

备份资料的考虑

  1. 造成系统销毁的问题-硬件问题

  2. 造成系统销毁的问题-硬件问题

  3. 造成系统损毁的问题-软件与人

  4. 主机角色不同,备份任务也不同

  5. 备份因素考虑

    备份哪些文件:哪些数据对系统或用户来说重要

    选择什么备份介置:使用硬盘还是网络文件系统

    考虑备份的方式:完整备份还是差异备份

    备份的频率：一天还是一周备份一次

    备份使用的工具：是利用tar、dd、dump等等备份共？

Linux需要备份的数据

操作系统本身需要备份文件
  • /etc/整个目录
  • /home/整个目录
  • /var/ spool/mail/
  • /var/spool/{atlcron}/
  • /boot/
  • /root/
  • 如果你自行安装过其他的软件，那么/usr/local/ 或/opt 也最好备份一-下!
网络服务的数据库
  • 软件本身的配置文件，例如: /etc/ 整个目录，/usr/local/ 整个目录
  • 软件服务提供的数据，以ww及Mariadb为例:www资料: /var/www 整个目录或/srv/www 整个目录，及系统的用户家目录Mariadb : /var/lib/mysql 整个目录
  • 其他在Linux 主机上而提供的服务之数据库文件!
推荐需要备份的目录
  • /etc:包含很多文件.许多网络配置文件也在/etc 中.
  • /home,/toot:用户家目录
  • /var/spool/mail/,/var/spool/cron/,/var/spool/at/
  • /var/lib
不需要备份的目录
  • /dev：这个随便要不要备份
  • /proc,/sys,/run：不需要备份
  • /mnt,/media：如果没有在目录内放置系统文件,无需备份
  • /tmp：临时缓存文件,不需要备份

备份种类、频率与工具选择

完整备份之增量备份(Incremental backup)

所谓的累积备份，指的是在系统在进行完第一-次完整备份后，经过一段时间的运作，比较系统与备份文件之间的差异，仅备份有差异的文件而已。而第二次累积备份则与第- -次累积备份的数据比较，也是仅备份有差异的数据而已。如此- -来，由于仅备份有差异的数据，因此备份的数据量小且快速!

#利用dd将/dev/sdb 备份到完成一模一样的/dev/sdc硬盘上:
[root@localhost /]# dd if=/dev/sdb of=/dev/sdc
记录了2097152+0 的读入
记录了2097152+0 的写出
1073741824字节(1.1 GB)已复制，43.212 秒，24.8 MB/秒
[root@localhost /]# mount /dev/sdc /data2/
[root@localhost /]# ls -dlH /data1/ /data2/
drwxr-xr-x 3 root root 4096 3月  10 22:01 /data1/
drwxr-xr-x 3 root root 4096 3月  10 22:01 /data2/
#注意不同文件系统采用dd也能备份

#使用pcio来备份还原整个系统
[root@localhost data3]# find /etc/ -print |cpio -covB > /dev/sdb
[root@localhost data3]# ls -l /etc/passwd
-rw-r--r-- 1 root root 2253 3月  10 22:14 /etc/passwd
[root@localhost data3]# rm -rf /etc/passwd
[root@localhost data3]# cpio -idu  < /dev/sdb 
73734 块
[root@localhost data3]# ls -l /etc/passwd
-rw-r--r-- 1 root root 2253 3月  10 22:16 /etc/passwd
>注意，cpio恢复的路径，如果cpio在打包备份的时候用的是绝对路径，那么在恢复的时候会自动恢复到这些绝对路径下，本例就会将备份文件全部还原到/etc路径下对应的目录中。同理，如果在打包备份用的是相对路径，还原时也将恢复到相对路径下。
通过上面的示例，可以看出，cpio无法直接读取文件，它需要每个文件或者目录的完整路径名才能识别读取，而find命令的输出刚好做到了这点，因此，cpio命令一般和find命令配合使用。

#使用dump将home完整备份要/backupfaata上
#1、完整备份
[root@localhost data2]#xfsdump -10  -L 'full' -M 'full' -f /data1/boot_dump /dev/sda1 
#2、第一次增量备份
[root@localhost data2]#xfsdump -11  -L 'full' -M 'full' -f /data1/boot_dump1 /dev/sda1 
##-f 备份到哪里 -L 数据标签 -M 设备标签
#3、恢复数据
[root@localhost data2]# xfsrestore -f /data1/boot_dump /data2/

#tar进行完整备份
[root@localhost mnt]#tar -jcvp -f /mnt/system.tar.bz2 /etc
[root@localhost mnt]#tar -jxvf system.tar.bz2 
[root@localhost mnt]#rm -rf /etc/passwd
[root@localhost mnt]# ls -l /etc/passwd
-rw-r--r-- 1 root root 2253 3月  10 22:16 /etc/passwd

完整备份之差异备份(Differential backup)

每次的备份都是与原始的完整备份比较的结果。

#使用tar -N来进行差异备份
[root@localhost /]# tar -N '2020-3-11' -jpcv -f /mnt/tar_2020-3-11.bz2 /home/
[root@localhost mnt]# tar -jxvf tar_2020-3-11.bz2

home/admin/

  • 异地备援:每月定期的将数据分别(a)刻录到光盘上面(b)使用网络传输到另-部机器上面。
